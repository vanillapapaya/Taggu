## 데이터 라이선스

### character_aliases.json
- 사용자 편집 가능. fan-curated.
- 잘못된 매핑/누락 발견 시 PR 환영.
- 원본 데이터 출처: Danbooru 태그명 + 한국어 위키 / 사용자 등록.

### 사용자 라이브러리
- 사용자가 인덱싱한 이미지 파일 = 사용자 자산
- DB(images.db), 임베딩, 학습된 centroid = 사용자 자산
- Yoink는 어떤 데이터도 외부로 전송 안 함 (단, 사용자가 API 모드 선택 시 그 provider에만)

## 개인정보 처리

### 로컬 모드 (기본)
- **데이터 외부 전송 0**
- 모델 추론, 임베딩, 검색 전부 사용자 PC에서
- 첫 실행 시 모델 다운로드를 위해 huggingface.co와 통신 — 모델 파일만, 사용자 데이터 X

### API 모드
- 선택한 provider(OpenAI/Anthropic/Gemini)의 서버로 **이미지 + 프롬프트 텍스트 전송**
- provider의 데이터 정책 적용:
  - OpenAI: 기본 학습 사용 안 함 (API 키 사용 시), 30일 보관 후 삭제
  - Anthropic: 기본 학습 사용 안 함, 30일 보관
  - Google Gemini: 무료 티어는 학습에 사용될 수 있음 (유료 키 권장)
- 각 provider의 최신 정책 확인 권장
- 민감/사적 이미지는 로컬 모드(GPU 또는 다운그레이드된 3B) 사용 권장

### API 키 저장
- 현재: `settings.json` 평문 저장 (gitignore)
- 향후: Windows Credential Manager로 마이그레이션 검토
- 사용자가 직접 파일 보호 (NTFS 권한, 디스크 암호화)

## 저작권 (사용자 책임)

Yoink는 도구이므로 사용자의 책임 영역:
- 인덱싱하는 이미지의 합법적 보유
- 저작물의 사적 정리 vs 공개 배포 구분
- API provider에 보낸 이미지가 트레이닝 데이터로 사용될 가능성 (위 참조)

Yoink 자체는 어떤 저작권 침해 행위도 권장/방조하지 않음.

## 보안

- 인증 없음 → LAN 노출 시 같은 네트워크의 누구나 접근 가능
- 가족/룸메이트 환경에선 신경 안 써도 됨
- 공공 WiFi/외부 노출은 절대 금지 (방화벽 inbound 차단 유지)
- API 키는 `.gitignore`에 있지만 PC 분실/도난 시 노출 가능

## 향후 (SaaS 버전 시)

[[00-프로젝트-개요]] 끝의 "Yoink가 SaaS로 가는 경우"는 별도 컴플라이언스 영역:
- 한국 정통망법 / GDPR / CCPA 적용
- 사용자 가입 시 약관 + 개인정보처리방침
- DMCA takedown 절차
- 성인 콘텐츠 분리 (Pixiv R-18 등 인덱싱 시 별도 카테고리)

현재 로컬 도구는 이 모두 해당 X (사용자 1인 환경).
