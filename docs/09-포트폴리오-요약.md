# Taggu — 포트폴리오 요약

**로컬에서 돌아가는 한국어 짤·이미지 라이브러리 자동 태깅 & 의미 검색 도구**

> 한 줄: 데이터 분석가로 일하면서, 수천 장 쌓인 내 이미지 폴더를 한국어로 검색하고 싶어 직접 만든 100% 로컬 AI 태깅·검색 앱.

성격: 개인 프로젝트 · 내 직무: 데이터 분석가 (개발은 독학) · 도구: Python · FastAPI · SQLite · 사전학습 AI 모델(WD14/CLIP/VLM) · AI 코딩 도구 활용

---

## 솔직한 전제 (이 문서를 읽는 분께)

저는 데이터 분석가입니다. 풀스택 개발자도, 머신러닝 엔지니어도 아닙니다. 이 프로젝트는 **제 실제 문제를 풀려고 새 기술을 학습하며 만든 첫 본격 결과물**이고, 막히는 부분은 AI 코딩 도구의 도움을 많이 받았습니다.

그래서 이 문서는 "내가 AI 시스템 전문가다"가 아니라 **"분석가가 문제를 정의하고, 도구를 비교·선택하고, 끝까지 돌아가는 걸 만들어냈다"**는 이야기입니다. 제가 자신 있게 설명할 수 있는 부분만 적었습니다.

---

## 문제 정의 (분석가가 가장 잘하는 부분)

수천 장씩 쌓인 짤·이미지·스크린샷 폴더는 사실상 검색이 안 됩니다. 기존 도구를 살펴보니 둘 중 하나였습니다.

- **Google/Apple 포토** — 자연어 검색은 되지만 클라우드 강제이고, 한국어·애니/짤에 약함.
- **호더용 도구(Hydrus 등)** — 로컬·프라이빗이지만 *태그를 정확히 입력해야만* 찾을 수 있음. 한국어 자연어 검색이 없음.

"**내 PC 안에 두면서** + **한국어로 대충 떠올린 말로 검색**"이 둘 다 되는 게 없었습니다. 그 빈칸을 직접 메우기로 했습니다.

---

## 내가 한 일

### 1. 문제에 맞는 모델을 비교하고 조합
하나의 모델로는 안 됐습니다. 각각 잘하는 게 달라서, **이미 공개된 모델 3개를 골라 조합**했습니다.

| 모델 | 쓴 이유 | 한계 |
|---|---|---|
| WD14 | 애니 캐릭터/작품 인식 | 한국어·자유 설명 못 함 |
| Qwen2.5-VL (또는 API) | 한국어 태그+설명 생성 | 캐릭터 정확도 약함 |
| CLIP | "비슷한 이미지" 유사도 | 글자 라벨이 없음 |

세 결과를 SQLite 한 곳에 모아 텍스트 검색·유사 이미지를 동시에 제공합니다. **모델 내부를 만든 게 아니라, 공개 모델을 평가해서 알맞게 배치한 것**이 제 기여입니다.

### 2. 비용 vs 품질을 직접 측정 (분석가식 접근)
로컬 GPU와 클라우드 API(OpenAI/Anthropic/Gemini) 중 뭘 쓸지 정하려고, **샘플 20장에 대해 모델별 태그 품질을 정성 평가하고 1000장 기준 비용을 추정**했습니다.

| 모드 | 1000장 비용 | 품질(주관 0~10) |
|---|---|---|
| 로컬 Qwen 7B | 전기료만 | 9.5~10 |
| Gemini 2.0 Flash | ~₩200 | 8.5 (가성비 1위) |
| OpenAI gpt-4o-mini | ~₩4,000 | 9 |

이 비교표 덕분에 "GPU 없으면 Gemini, 있으면 로컬"이라는 명확한 사용 가이드가 나왔습니다.

### 3. GPU가 있든 없든 돌아가게
같은 앱이 좋은 GPU에서는 로컬 모델로, GPU 없는 노트북에서는 API로 동작하도록 백엔드를 바꿔 끼울 수 있게 만들었습니다. 설정 화면에서 API 키를 한 번 테스트해보고 저장합니다.

### 4. 막힌 곳을 찾아서 우회 (학습 능력)
로컬 모델이 그래픽카드 메모리를 너무 많이 먹어서 안 돌아갔습니다. 찾아보니 `autoawq`라는 방법은 제 환경(Windows)에서 설치가 안 됐고, 대신 `bitsandbytes`라는 대안으로 **메모리 사용량을 약 16GB → 5.5GB로 줄여** 보급형 GPU에서도 돌게 만들었습니다. 원리를 깊이 아는 건 아니지만, 문제를 정의하고 대안을 찾아 적용했습니다.

## 결과물

- **한국어 자연어로 의미 검색** ("라프텔에서 본 그 짤")
- 텍스트 검색 + 비슷한 이미지 찾기 + 캐릭터 인식
- **로컬/클라우드 한 앱 지원** — GPU 유무와 무관
- **100% 내 PC에서 동작**, 설치는 EXE 더블클릭
- 드래그로 태그 정리, 즐겨찾기/숨김, 일괄 작업 같은 실사용 UX

| | Google Photos | Hydrus | **Taggu** |
|---|---|---|---|
| 한국어 의미검색 | △ | ✕ | **◎** |
| 애니 캐릭터 인식 | ✕ | ◎ | **◎** |
| 100% 로컬 | ✕ | ◎ | **◎** |
| GPU 없이 동작 | ◎ | △ | **◎** |

## 이 프로젝트로 보여주고 싶은 것

- **문제를 정의하고 끝까지 만든다** — 불편을 느끼고, 기존 도구를 조사하고, 직접 동작하는 결과물까지.
- **데이터로 결정한다** — "느낌"이 아니라 비용·품질을 측정해서 모델을 골랐다.
- **모르는 분야도 학습해서 적용한다** — AI 모델, 웹 서버, 패키징 모두 이번에 처음 다뤘다.

## 솔직한 한계

- 개발 입문 단계라 AI 코딩 도구의 도움을 많이 받았다.
- Windows 전용, 단일 사용자(인증 없음).
- 애니/일러 위주라 실사 사진에는 캐릭터 인식이 약하다.
