# 02 — 데이터 파이프라인

## 인덱싱 (새 폴더 추가)

```mermaid
flowchart TD
    A["사용자: 폴더 경로 입력"] --> B["POST /api/index"]
    B --> C{"index_lock 확보?"}
    C -->|아니오| D["409 - 다른 작업 중"]
    C -->|예| E["background thread run_indexing"]
    E --> F["scan_images: 폴더 재귀 + 라이브러리 폴더 제외"]
    F --> G{"각 이미지 mtime 비교"}
    G -->|기존과 동일| H[skip]
    G -->|새 or 변경| I["처리 큐에 추가"]
    I --> J["PIL.open + load() — 디스크 디코드 1회"]
    J --> K["3개 모델 PIL 공유"]
    K --> L["CLIP 임베딩 (512d)"]
    K --> M["WD14: 캐릭터 영문 → alias 한국어"]
    K --> N["CCIP: 768d 동일성 임베딩"]
    K --> O["VLM (옵션): 한국어 태그/설명"]
    L --> P["INSERT/UPDATE images 행"]
    M --> P
    N --> P
    O --> P
    P --> Q{"큐 비었나?"}
    Q -->|아니오| J
    Q -->|예| R[done]
```

**최적화 포인트:**
- 라이브러리 폴더(`*.library`, `*.eagle`, `*.aseprite-cache`, `*.thumbs`) 사전 제외 — Eagle 같은 앱이 사본 복사한 거 중복 인덱싱 방지
- PIL `Image.open + .load()` 1회 → 3개 ONNX/torch 모델이 같은 객체 재사용 → 디스크 디코드 비용 절약
- mtime 변경 없으면 skip — 점진적 인덱싱

## 검색 (텍스트)

```mermaid
flowchart LR
    A["검색어 입력"] --> B["GET /api/search?q="]
    B --> C["q_terms = 소문자 split"]
    C --> D{"각 이미지 row"}
    D --> E["user_haystack + other_haystack lower-join"]
    E --> F{"모든 q_terms가 매치?"}
    F -->|user에 다 있음| G["boost = 0.7"]
    F -->|어디든 다 있음| H["boost = 0.5"]
    F -->|아니| X["제외"]
    G --> I["score = CLIP cosine + boost"]
    H --> I
    I --> J["score 내림차순 정렬, 상위 limit개"]
```

**규칙:**
- 띄어쓰기 = AND (모든 단어 매치 필수)
- 내 태그(`user_tags`) 매치 시 다른 필드보다 가중치 ↑
- CLIP 유사도(0~1) + 텍스트 boost를 합한 score로 정렬

## 비슷한 이미지 (CLIP cosine)

```mermaid
flowchart LR
    A["모달 ≈ 클릭"] --> B["GET /api/similar/(id)"]
    B --> C["query_emb = images.id의 clip_embedding"]
    C --> D["모든 이미지 emb 로드"]
    D --> E["코사인 유사도 계산 (L2-norm dot product)"]
    E --> F["상위 K개 반환 (자기 자신 제외)"]
```

200~5000장 규모에선 Python 루프로도 100ms 이내. 수만 장 이상이면 sqlite-vec / FAISS 도입 검토 (현재 미적용).

## 캐릭터 학습 (CCIP centroid)

### 학습 (사용자 행동 자동)

```mermaid
sequenceDiagram
    participant U as User
    participant F as Frontend
    participant B as Backend
    participant D as SQLite
    U->>F: AI 태그를 char로 드래그<br/>또는 캐릭터 칩 더블클릭 편집
    F->>B: POST /api/image/{id}/move_tag<br/>{tag, source, target:char}
    B->>D: SELECT ccip_embedding, wd_chars_ko
    B->>B: before/after diff → 신규 캐릭터 식별
    B->>D: char_prototypes에서 기존 centroid 조회
    B->>B: 신규 centroid = (old*count + emb) / (count+1)<br/>L2 normalize
    B->>D: INSERT OR REPLACE char_prototypes<br/>(name, centroid, count+1, now)
    B->>D: UPDATE images SET wd_chars_ko=?
    B-->>F: 변경 결과 반환
```

### 추론 (모달 진입 시 자동)

```mermaid
flowchart LR
    A["모달 열림"] --> B["GET /api/image/{id}/suggest_chars"]
    B --> C["이미지 ccip_embedding 로드"]
    C --> D["모든 char_prototypes centroid 로드"]
    D --> E["cosine 계산"]
    E --> F{"score >= 0.70?"}
    F -->|예| G["후보로 추가 (이미 단 캐릭터 제외)"]
    F -->|아니| X[skip]
    G --> H["상위 N개를 추천 칩으로 표시"]
    H --> I{"사용자 클릭?"}
    I -->|예| J["field_tags 호출 → centroid 재강화"]
    I -->|아니| K["무시 (학습 안 함, false positive 방지)"]
```

**임계값 근거:**
- CCIP 모델 메타데이터의 OPTICS eps = 0.18 (cosine distance) → cosine similarity ≥ 0.82 = 동일 캐릭터 강한 신호
- 실용적으론 0.70부터 추천 노출 (사용자가 판단)
- 자동 부착(write) 임계값 0.85+ 향후 검토

## 태그 이동 (atomic, 6방향)

```mermaid
sequenceDiagram
    participant U as User
    participant F as Frontend
    participant B as Backend
    participant D as SQLite
    U->>F: AI 태그를 char 자리로 드래그
    F->>F: source='ai', target='char'
    F->>B: POST /api/image/{id}/move_tag {tag, source, target}
    B->>D: SELECT user_tags, wd_chars_ko, tags
    B->>B: src 필드에서 tag 제거<br/>dst 필드에 tag 추가 (중복 제거)
    B->>D: UPDATE images SET {src}=?, {dst}=?
    B->>B: target='char'이면 _learn_chars hook 발화
    B->>D: char_prototypes 업데이트 (centroid online mean)
    B-->>F: 3 필드 갱신된 배열 반환
    F->>F: 그리드 + 모달 동시 갱신
```
