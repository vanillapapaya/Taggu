# 프론트엔드 명세

## 기술 스택
- **Next.js 14+** (App Router)
- **Tailwind CSS**
- **Zustand** (상태 관리)
- **Swiper.js** (피드 스와이프)
- **next/image** (이미지 최적화)

## 페이지 구조

```
/                    → 메인 (검색 + 인기 짤)
/search?q=...        → 검색 결과
/feed                → 개인화 피드 (숏폼 스와이프)
/feed/trending       → 인기 트렌딩 피드
/image/{id}          → 이미지 상세
/creator/{id}        → 크리에이터 프로필
/upload              → 이미지 업로드
/login               → 로그인
/my/likes            → 내 좋아요 목록
```

## 모바일 우선 설계

- 기본: 모바일 (375px 기준)
- 반응형: 태블릿 (768px), 데스크톱 (1024px)
- 피드 스와이프는 모바일에서 핵심 경험
- 데스크톱에서는 피드를 카드 형태 + 키보드 방향키로 전환

## 성능 요구사항

| 항목 | 목표 |
|------|------|
| 검색 결과 첫 렌더 | 500ms 이내 |
| 피드 스와이프 전환 | 100ms 이내 (프리로딩) |
| 이미지 로드 (썸네일) | 200ms 이내 (CDN) |
| LCP (Largest Contentful Paint) | 2.5s 이내 |
| CLS (Cumulative Layout Shift) | 0.1 이하 |

## 이미지 프리로딩 전략

```
현재 보고 있는 짤: N
프리로드 대상: N+1, N+2, N+3 (앞 3장)

구현:
- next/image의 priority 속성 활용
- IntersectionObserver로 스크롤 위치 감지
- 피드 API를 20개씩 배치 호출, 잔여 5개 이하면 다음 배치 요청
```
