# 04 — 프론트엔드 명세

## 기술 스택

- **Vanilla HTML/CSS/JS** — 프레임워크 없음, 빌드 없음
- 단일 파일: `templates/index.html` (~3000줄)
- Jinja2로 서버 사이드 변수 1개만 주입: `is_local` (현재 머신 요청 여부)
- 의도적 선택: Ctrl+F5 한 번에 UI 변경 즉시 반영, 빌드 단계 0

## 화면 구성

### 헤더 (한 줄 강제 유지)

```
[검색박스(분홍)] [전체|★|숨김] [필터▾]     [⋯ ⚙ ? ⌨ ℹ] [▤ ↻]
                                                       ^dev-only
```

- 좁아져도 `flex-wrap: nowrap`. 검색 입력만 줄어듦 (`flex: 1 1 0; min-width: 0`)
- 검색 버튼 = 분홍(`--char`), 랜덤 = 노랑(`--accent`), 홈(⌂) = 투명+보더
- 헤더 우측 아이콘:
  - `⋯` 관리 도구 토글
  - `⚙` AI 설정 모달
  - `?` 처음 사용 가이드
  - `⌨` 단축키 popover
  - `ℹ` 정보/통계 popover
  - `!` 에러 알림 (에러 발생 시만 표시, 빨강)
  - `▤` 로그 파일 열기 (dev mode only)
  - `↻` 서버 재시작 (dev mode only)

### 필터 row (header 아래, 토글로 표시/숨김)

```
필터  [캐릭터 없음] [내 태그 없음] [AI 태그 없음]  [초기화]
```

- 기본: 숨김 (`localStorage.taggu.filterBar = 'hidden'`)
- 헤더 [필터▾] 버튼으로 토글
- 활성 필터 있으면 자동 표시 (가렸어도 안전 — 어떤 필터 적용 중인지 보장)
- 좁은 창에서 가로 스크롤 (flex-wrap: nowrap, overflow-x: auto)

### 관리 도구 (⋯, 2줄)

```
[폴더 경로 입력         ] [폴더 고르기] [새 이미지 추가]
[캐릭터 인식] [AI 분석] [폴더 목록]              [유지보수 ▾]
```

유지보수 드롭다운:
- 새 캐릭터 자동 번역
- 이름 다시 적용 (사용자 편집 보존)
- 중복 태그 정리
- 라이브러리 중복 청소
- 기존 사진 학습 준비 (CCIP backfill)

### 카드 그리드

- `grid-template-columns: repeat(auto-fill, minmax(240px, 1fr))`
- 카드 hover 시 등장:
  - 좌상: `★` 즐겨찾기, `✕` 숨김 토글
  - 우상: `복사` 버튼
  - 하단: 파일명 + (검색 시) 일치율 %
- 카드 안 태그 칩: 클릭 시 해당 태그로 즉시 검색
- 카드 클릭: 모달 열기
- Ctrl/Shift + 클릭: 다중 선택 (bulk-bar 활성화)

### 모달

```
┌──── 이미지 (원본) ─────┬─── 정보 패널 ────────┐
│                        │ 파일명                │
│                        │ [복사] [★] [✕] [≈ 비슷]│
│                        │                       │
│    원본 이미지         │ 캐릭터                │
│                        │ [꽉×] [사야×]         │
│                        │ ┌ 추천 (학습됨) ────┐ │
│                        │ │ [히마리 87%]      │ │
│                        │ │ [코코로 76%]      │ │
│                        │ └──────────────────┘ │
│                        │                       │
│                        │ 내 태그               │
│                        │ [라프텔×] [+ 입력란]  │
│                        │                       │
│                        │ AI 태그               │
│                        │ [고양이×] [밈×] ...   │
│                        │                       │
│                        │ AI 설명               │
│                        │ "놀란 표정의..."      │
└──── ← (이전) ─ → (다음) ──── × (Esc로 닫기) ──┘
```

- 각 태그 칩:
  - 더블클릭 → contentEditable 인라인 편집
  - 드래그 → 다른 색 자리로 이동 (6방향 atomic)
  - `×` 클릭 → 제거
- 모달은 viewport portrait 시 이미지 위, 정보 아래로 stack

### Bulk Bar (다중 선택 시 하단 떠오름)

```
3개 선택됨  [★ 즐겨찾기] [★ 해제] [✕ 숨기기] [↺ 표시] [+ 태그 추가] [선택 해제]
```

`position: fixed; bottom: 24px; left: 50%; transform: translateX(-50%)` — pill 모양.

## 상태 관리

### 클라이언트 상태 (단순 JS 모듈 변수)

| 변수 | 의미 |
|---|---|
| `currentMode` | `'browse' | 'search' | 'random' | 'similar'` |
| `currentView` | `'all' | 'favorite' | 'hidden'` |
| `currentQuery` | 마지막 검색어 |
| `browseOffset` | browse 페이지네이션 |
| `cardData` | 현재 그리드의 row 배열 (모달에서 ←/→로 순회) |
| `modalIndex` | 모달 활성 인덱스 |
| `selected` | 다중 선택된 id Set |
| `filterFlags` | `{no_char, no_user, no_ai}` boolean |
| `errorLog` | 최근 20개 (logError로 누적) |
| `pollTimer` | `{ stopped: boolean }` — 재귀 setTimeout 토큰 |
| `cardImageObserver` | IntersectionObserver (offscreen src 해제) |

### 영속화 (localStorage)

| 키 | 값 |
|---|---|
| `taggu.devMode` | `'true' | 'false'` — 헤더 dev 아이콘 표시 |
| `taggu.toolbar` | `'show' | 'hide'` — 관리 도구 펼침 상태 |
| `taggu.filterBar` | `'visible' | 'hidden'` — 필터 row |
| `taggu.guideSeen` | `'true'` — 가이드 자동 표시 여부 |

## 인터랙션 디테일

### Offscreen 이미지 해제

- `<img data-thumb-src="..." loading="lazy">`로 시작 (src 없음)
- `IntersectionObserver(rootMargin: 600px)` 인스턴스 1개로 모든 카드 img 관찰
- 뷰포트 진입 → src 복원
- 뷰포트 이탈 → src 제거 (애니메이션 정지 + 디코더 해제)
- GIF/animated WebP 많은 그리드에서 CPU/배터리 절약 큼

### 에러 로깅

- 글로벌 `logError(msg, context)` — 최근 20개 in-memory
- `window.error` / `unhandledrejection` 자동 캐치
- `alert()` 후킹 — 메시지에 "오류/실패/에러" 포함되면 자동 누적
- 헤더 빨간 `!` 배지 (에러 0개일 땐 숨김, 발생 시 카운트)
- 클릭 → popover로 timestamp + msg + context 표시 + [비우기]

### 학습 추천 (모달 진입 시)

```js
loadCharSuggestions(imageId)
  ↓ GET /api/image/{id}/suggest_chars?min_score=0.70
  ↓ box.innerHTML = 칩들
  ↓ 클릭 → acceptSuggestion()
       → POST /api/image/{id}/field_tags (field=wd_chars_ko, tags=[...+name])
       → renderModalBody 재호출 (cardData 갱신)
       → 백엔드에서 _learn_chars hook 발화 → centroid 재강화
```
