**목표**: 만들기 전에 사람들이 원하는지 확인

**작업**:
- [ ] Fake door 랜딩페이지 제작 (Next.js + Vercel)

  - 데모 GIF: 실제 검색하는 것처럼 보이는 목업
- [ ] DC/에펨코리아/트위터에 홍보 게시글 작성
  - "이런 서비스 있으면 쓸 거임?" + 랜딩페이지 링크
- [ ] 반응 수집 (이메일 가입 수, 댓글 반응)

**Go/No-Go 기준**:
- 1주 내 이메일 가입 100명 이상 → Go
- 50-100명 → 피벗 고려
- 50명 미만 → 방향 재검토

**비용**: $0
**산출물**: 랜딩페이지, 반응 데이터

## Phase 1: 데이터 수집 + 인프라 (2주)

**목표**: seed data 1,000개 + 기본 인프라 구축

### Week 1: 인프라
- [ ] Railway 프로젝트 생성 (FastAPI + PostgreSQL + Redis)
- [ ] Cloudflare R2 버킷 생성
- [ ] DB 스키마 생성 (images, creators, users, user_interactions)
- [ ] Qdrant 컬렉션 생성 (clip_vectors, text_vectors)
- [ ] CLIP 모델 로딩 + 벡터화 파이프라인 구현
- [ ] e5-large 모델 로딩 + 텍스트 벡터화 파이프라인 구현
- [ ] 이미지 처리 파이프라인 (다운로드 → 리사이즈 → R2 업로드)

### Week 2: 데이터 수집
- [ ] 브라우저 확장 프로그램 개발 (Chrome Extension)
  - 이미지 위 클릭 → 메타데이터 추출 → 서버 전송
- [ ] 글로벌 밈 수동 수집 500개 (기획자)
- [ ] 한국 커뮤니티 짤 수동 수집 500개 (기획자)
- [ ] 수집된 이미지 → 벡터화 배치 실행

**비용**: $5 (캡셔닝 필요한 이미지만)
**산출물**: seed data 1,000개, 벡터 인덱스, 브라우저 확장
