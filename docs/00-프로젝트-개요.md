# Yoink - 프로젝트 개요

## 서비스 한 줄 정의
**짤과 원작자를 연결하는 검색 엔진 + 짤 피드 플랫폼**

## 핵심 유스케이스
커뮤니티 하다가 "아 그 짤 뭐였지?" 할 때 자연어로 찾을 수 있는 서비스.
숏폼 스와이프 피드로 추천 짤을 탐색하고, 원작자를 찾아 연결해주는 플랫폼.

## 핵심 기능 3가지

### 1. 짤 검색 (유입 채널)
- 자연어 텍스트 → 이미지 검색 ("뒤돌아보면서 웃는 남자")
- CLIP/SigLIP 벡터(시각 유사도) + 캡션/태그 텍스트 벡터(의미 유사도) 하이브리드
- 검색 결과: 썸네일 + 원본 출처 링크 + 작가 정보

### 2. 짤 피드 (리텐션 채널)
- 숏폼 스와이프 UI (틱톡/인스타 릴스 UX)
- 유저 행동(검색, 좋아요) 기반 벡터 추천
- N장마다 광고 삽입 (인터스티셜)
- 다양성: 추천 70% + 랜덤 탐색 30%

### 3. 창작자 연결 (확장 채널)
- 원작자 프로필 페이지 (Pixiv/X 계정 OAuth 연동)
- "이 짤의 원본은?" → 출처 추적
- 장기: 커미션 연결, 프로모션 광고, AI vs 원본 감별

## 수익 모델
| 모델 | 시점 | 설명 |
|------|------|------|
| 피드 인터스티셜 광고 | 초기 | 5장마다 광고 1개, DAU 3,000-5,000이면 서버비 커버 |
| 창작자 프로모션 | 중기 | 창작자가 노출 구매 |
| 프리미엄 프로필 | 중기 | 포트폴리오 상단 노출, 통계 |
| 커미션 수수료 | 장기 | 거래 연결 시 수수료 |

## 경쟁 분석 요약
- 검색이 강한 서비스(2짤, SauceNAO)는 피드가 없음
- 피드가 강한 서비스(9GAG, iFunny)는 검색이 약함
- **검색 + 피드 + 창작자 연결을 동시에 갖춘 서비스는 현재 없음**
