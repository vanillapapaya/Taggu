# 00 — 프로젝트 개요

## 한 줄 정의

**한국어 우선 로컬 이미지 라이브러리 자동 태깅 & 의미 검색 도구.** GPU에서 직접 돌리는 VLM + CLIP + 캐릭터 인식, 또는 OpenAI/Anthropic/Gemini API로 GPU 없이도.

## 어떤 문제를 푸는가

오랫동안 모아둔 짤·일러스트·스크린샷이 폴더에 수천 장 쌓여 있는데:
- 파일명은 무의미 (`KakaoTalk_20240315_001.jpg`)
- 어디에 뭘 모아뒀는지 기억 안 남
- "그 짤 어디 갔지?" 한 시간 헤매다 포기
- 클라우드 서비스(Google Photos 등)에 올리기엔 프라이버시/저작권 거슬림

**Yoink는 이걸 한 번의 인덱싱으로 자동 태깅하고, 한국어 자연어로 검색 가능하게 만든다.**

## 핵심 기능 3가지

### 1. 자동 태깅 (3계층)
- **WD14 EVA02-Large** (ONNX, CPU): 애니/만화 캐릭터 영문 태그 추출 → `character_aliases.json`으로 한국어 매핑
- **Qwen2.5-VL** 또는 **API LLM** (GPU/API): 한국어 자유형 태그 5~10개 + 한 줄 설명
- **CLIP ViT-B/32** (CPU/GPU): 512d 시각 임베딩 — 의미 검색 + 유사 이미지

### 2. 한국어 의미 검색
- 텍스트 검색: 한국어 IME 안전, 띄어쓰기 = AND 다중 태그
- 비슷한 이미지: CLIP 코사인 유사도
- 랜덤 탐색: R 키 / [랜덤] 버튼
- 미분류 필터: 캐릭터/내 태그/AI 태그 없는 사진만

### 3. 개인 캐릭터 학습 (CCIP)
- 사용자가 캐릭터 칩을 편집/드래그할 때마다 **자동 누적 학습**
- WD14가 못 잡는 2024년 이후 신규 캐릭터를 사용자 라벨로 점진적 학습
- 새 이미지에 "추천 (87%)" 자동 등장

## 설계 원칙

| 원칙 | 의미 |
|---|---|
| **로컬 우선** | DB·이미지·인덱스 전부 사용자 디스크. 클라우드 의존 0 |
| **한국어 일관성** | UI, 태그, 에러 메시지, 단축키, 가이드 전부 |
| **단일 사용자** | 인증/공유/멀티테넌트 X. 본인 PC에서 본인이 씀 |
| **점진적 인덱싱** | 폴더 다시 추가하면 mtime 비교로 변경분만 처리 |
| **사용자 라벨 = 신호** | 수동 편집은 보존 + 학습 데이터로 누적 |

## 누가 쓰나

타겟 사용자상:
- 한국어 사용자
- 짤/일러스트/스크린샷을 폴더 단위로 모아두는 습관
- 사용자 라이브러리 200~10,000장 규모
- 프라이버시 의식 (클라우드 업로드 거리감)
- 윈도우 데스크톱 (현재 macOS/Linux 미지원)
- 옵션: NVIDIA GPU 보유 (없으면 API 모드)

비타겟:
- 사진 백업/공유가 주 목적 (→ Google Photos)
- 협업/공유 필요 (→ Pixiv collections, Pinterest)
- 모바일에서 주로 사용 (현재 데스크톱만)

## 비교 분석

| 도구 | 강점 | Yoink와의 차별점 |
|---|---|---|
| **Eagle** | 폴더/태그 관리 매우 강함, 유료 | Yoink는 자동 태깅, Eagle은 수동 |
| **Hydrus Network** | 무료/오픈, 태그 시스템 성숙 | Yoink는 한국어, Hydrus는 영어 중심 |
| **Google Photos** | 무료/모바일, 얼굴 인식 | Yoink는 로컬, 캐릭터 인식 특화 |
| **PhotoPrism / Immich** | 셀프호스트 사진 백업 | Yoink는 짤/일러스트 특화, 단일 사용자 |
| **SauceNAO / iqdb** | 역검색 (출처 찾기) | Yoink는 본인 라이브러리 정리 |

Yoink의 차별 포지션: **"한국어 + 애니/일러스트 + 로컬 + 자동 태깅"** 4-tuple. 메이저 도구 없는 자리.
